# D / V — прототип

Реализация прототипа из GDD: дешёвый честный срез петли **«канат тьмы + экономика хила»**,
чтобы ответить на единственный вопрос первого среза (§12) —

> **Перетягивание каната само колеблется вокруг центра — или сваливается?**

Визуал намеренно примитивен (фигуры-плейсхолдеры): сначала крепкая геймплейная основа,
красота — потом.

---

## Что внутри (по срезам)

**Срез 1 (минимум §12), подтверждён:** 1 D + 1 V, враги **из центра**, **шкала тьмы**
от вложенных очков, **хил D** за валюту, **крахи** §6, телеметрия §12.

**Срез 2 (шаг к игре §7, §2, §3):**
- **Толпа:** настраиваемые **N D + M V** по углам (§7, дефолт 6/6).
- **Второй тип врага — толстяк** (§3): тяжёлый, медленный, за **потолком V по толщине**
  (§2). V почти не жжёт его — только **ставит метку**; D бьёт по меченому ×2.4 (вклад V =
  умноженный выстрел D, §2). Это делает D структурно необходимым и хоронит «двойку V в углу».
- **Состав** врагов (§3): доля толстяков растёт с прогрессом сессии (ось «что приходит»),
  отдельно от тьмы (ось «интенсивности»).

Нет (намеренно): Пульса D, площадной ветки V, сайдгрейдов, ачивок, дальнобоя/охотника/глушителя.

---

## Запуск

### 1. Headless-эксперимент (главный инструмент — «три кубика за вечер», §12)
Прогоняет много сессий в Node, классифицирует форму каната, печатает sparkline.

```bash
cd prototype
npm run sim                                  # дефолт: толпа 6/6, жадный D
node tools/run-sim.js --set ai.dDarkStopAt=0.6           # отзывчивый D
node tools/run-sim.js --set session.numD=8 --set session.numV=3   # перекос популяции
node tools/run-sim.js --set spawn.fatChanceMax=0.6       # больше толстяков
```

Свип любого числа из `src/config.js` без правки файла:
```bash
node tools/run-sim.js --set economy.healPayPerHP=2 --set spawn.baseInterval=0.7 --csv=run.csv
```
Флаги: `--sessions=N --seconds=T --seed=S --sample=I --verbose --csv[=path] --set путь=значение`.

**Чтение вердикта (§12):**
- `OSCILLATES` / `STABLE` / `BOUNDED` — канат держится у центра → **есть игра**, контент сверху.
- `DRIFT→DARK` / `DRIFT→LIGHT` — монотонно сваливается в край → крутить `config.js`.
- «концовка» сессии (выжили / погибли-все-V / погибли-все-D) — **контекст, а не вердикт**:
  крах — естественный конец сессии (§7), он сам по себе не говорит о форме шкалы.

### 1b. Мета: выбор фракции между сессиями (§5, §7, §9, §10)
Над сессией: пул из ~20 игроков переигрывает серию, между сессиями выбирая фракцию по
подушевому доходу. Сходится ли популяция к балансу — или сваливается в одну фракцию?

```bash
npm run meta:landscape                       # подушевой доход D/V по раскладам D:V
npm run meta                                 # петля выбора (только стимулы)
node tools/run-meta.js --dynamics --set meta.joinRuleStrength=1   # + правило входа §7
```
- `landscape` отвечает: возникает ли «премия за дефицит» §5 (меньшинство доходнее)?
  Колонка `V:хил%` — доля дохода V с ХИЛА (а не off-role убийств).
- `dynamics` отвечает: `CONVERGE→BALANCE` (самокоррекция работает) или `COLLAPSE→all-D/V`.

### 2. Браузер (играбельная основа)
ES-модули требуют HTTP (не `file://`):
```bash
cd prototype
npm run serve            # python3 -m http.server 8080
# открыть http://localhost:8080
```
Боты ведут обе стороны. «Вселиться»: `1`→D, `2`→V, `0`→отпустить. Движение `WASD`,
выстрел — зажатая мышь (у V выстрел = хил союзника / урон врагу). Тумблер «отзывчивый D»
и график **net(t)** показывают разницу «дрейф против плато» вживую.

## Архитектура

Ключевое решение: **детерминированное сим-ядро (`src/sim`, чистый JS без DOM) отделено
от рендера.** Одно и то же ядро бежит и headless в Node (быстрый прогон каната), и в
браузере на Canvas.

```
src/
  config.js         все числа = placeholder'ы (§11: "насколько" решается прототипом)
  core/             vec2, сид-RNG (детерминизм прогонов)
  sim/              ЯДРО (переносимо): world(толпа §7), entities(рой+толстяк),
                    darkness(канат §4), economy(§5), spawner(состав §3),
                    combat(метка §2, типы §3), ai(боты D/V/врагов)
  telemetry/        recorder + классификатор формы каната (§12)
  meta/             factionMeta — петля выбора фракции между сессиями (§5,§7,§9,§10)
  render/           renderer + hud (только браузер, примитивные фигуры)
  browser-main.js   оболочка: цикл, ввод, вселение
tools/run-sim.js    headless-раннер сессии (форма каната, свипы)
tools/run-meta.js   мета-раннер (ландшафт дохода + динамика выбора фракции)
```

Каждый модуль помечен ссылками на разделы GDD — причины важнее кода (как и просит документ).

## Результаты прогонов (честно)

**Решающий рычаг каната — не числа, а ОТЗЫВЧИВОСТЬ вложений:**

- **Жадные** боты (вкладывают очко сразу, как накопят) → канат **монотонно сваливается
  в тьму** во ВСЕХ конфигурациях. Доход D с убийств стабилен и rate-capped; бонус тёмного
  мира — **симметричный** множитель (ускоряет лидера, не меняет баланс). Подушевая скорость
  вложения D в тьму так велика, что **даже 3 D перетягивают 8 V** — перекос популяции
  направление не разворачивает, а доход V структурно ограничен тем, сколько урона вообще
  получают D.
- **Отзывчивый** D (придерживает тьму у сетпоинта — иллюстрация §4: «взять огромную ноду
  урона, понимая, что она кладёт моих V под давление?») → канат **держится у центра**
  (`STABLE`/`BOUNDED`): net в десятках, не в сотнях, пара живёт всю сессию.

```
ДЕФОЛТ (жадный, 6/6):             ОТЗЫВЧИВЫЙ (--set ai.dDarkStopAt=0.6):
net │ ...:::---===+++***###@        net │ ..--==~~ (узкая полоса у центра)
    DRIFT→DARK,  net→300+               BOUNDED,  net∈[-12,23]
```

**Вывод:** ядро *способно* удержать канат, но не само по себе — **только если стимулы
реально направляют вложения** (ставка §10). Центральный риск §12 уточнён: с «сходится ли
математика» на «заставит ли своекорыстный расчёт D *притормозить* затемнение».

**Толстяк и метка (§2) работают по построению:** burn V по толстяку ≈ 0.9/выстрел против
300 hp → V один его **не добьёт никогда**; D по меченому — за ~6 выстрелов. Потолок V и
необходимость D закодированы в механике, не в подкрутке.

### Петля выбора фракции (§5, §7, §10) — главная находка

`meta:landscape` измерил подушевой доход по раскладам D:V. Результат резкий:

```
  D/ V | доход D | доход V | V/D  | V:хил%
  2/18 |   408   |    9    | 0.02 |   0%      ← дефицитный D доходен ЕЩЁ сильнее
 10/10 |    53   |   26    | 0.50 |   0%
 16/ 4 |    49   |   35    | 0.71 |   0%      ← пик V/D, но < 1: V так и не обгоняет D
```

1. **«Премия за дефицит» §5 НЕ возникает.** D доходнее V на ВСЕХ раскладах (V/D макс 0.71).
   Премия есть по *направлению* (V/D растёт при дефиците V), но слаба по *магнитуде*.
2. **Корень: `V:хил% = 0%` везде.** Доход V на 100% из off-role убийств роя, а не из хила
   (§8 «побег из роли»). Тюнинг `healPayPerHP` бесполезен — хил-дохода просто нет.
3. **Почему:** одноцелевой хил-снаряд V в толпе **перехватывается врагами** на пути к D
   (позиционная проблема §2 в пределе). Проверено точечно: без помех хил начисляется —
   значит 0% в толпе structural, не баг. → **площадная ветка V (§2) — не вкус, а
   необходимость экономики**: «что внутри радиуса своё — лечится» обходит блокировку.
4. **Динамика выбора:** на одних стимулах популяция **`COLLAPSE→all-D`** (fD→0.85, 19/1).
   **Правило входа §7** («нельзя к перевешивающей стороне», `meta.joinRuleStrength=1`)
   **спасает баланс**: `CONVERGE→BALANCE` 4/4 (держит ~11/9). Корректор баланса популяции —
   правило входа, а не стимулы (пока хил-экономика не чинит премию).

## Куда копать дальше (привязка к §11)

- **Чинить хил-экономику (приоритет):** ввести **площадную ветку V** (§2) — хил сквозь толпу
  не блокируется; тогда `V:хил%` вырастет, появится премия за дефицит, и выбор фракции сможет
  сходиться на стимулах, а не только на правиле §7.

- **§11.1 (сходимость):** отзывчивость даёт плато/полосу — нужен поиск живого *колебания*
  (запаздывания вложений, гистерезис §11.5).
- **§11.2 (пропускная способность хила):** после площадной ветки — свип `V.healPower` ×
  `enemy.*DarkGain`; сделать толстяка реальной «стеной», кормящей экономику хила.
- **§11.4 (сжатие V-кромки):** ввести дальнобоя и охотника (идёт на хилера).
- **§11.7 (доступность фракций):** мета-петля уже на месте — крутить `meta.*`
  (`beta`, `vAttract`, `joinRuleStrength`) для политики входа §7/§9.

Все рычаги — в `src/config.js`, прогоняются через `--set`.
