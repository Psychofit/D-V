# D / V — прототип (первый срез)

Реализация **минимального прототипа** из GDD (раздел 12): дешёвый честный срез петли
**«канат тьмы + экономика хила»**, чтобы ответить на единственный вопрос среза —

> **Перетягивание каната само колеблется вокруг центра — или сваливается?**

Визуал намеренно примитивен (фигуры-плейсхолдеры): сначала крепкая геймплейная основа,
красота — потом.

---

## Что внутри среза (и чего нет)

Есть (минимум по §12): **1 D + 1 V**, враги **из центра**, **шкала тьмы** от вложенных
очков, **хил D**, за который V получает валюту, **крахи** §6, телеметрия §12.

Нет (намеренно, по §12): сайдгрейдов, Пульса, ачивок, открывашек, второго типа врага.

---

## Запуск

### 1. Headless-эксперимент (главный инструмент — «три кубика за вечер», §12)
Прогоняет много сессий в Node, классифицирует форму каната, печатает sparkline.

```bash
cd prototype
npm run sim                          # дефолт: жадный D
node tools/run-sim.js --sessions=8 --seconds=300
node tools/run-sim.js --set ai.dDarkStopAt=0.6   # отзывчивый D
```

Свип любого числа из `src/config.js` без правки файла:
```bash
node tools/run-sim.js --set economy.healPayPerHP=2 --set spawn.baseInterval=0.7 --csv=run.csv
```
Флаги: `--sessions=N --seconds=T --seed=S --sample=I --verbose --csv[=path] --set путь=значение`.

**Чтение вердикта (§12):**
- `OSCILLATES` / `STABLE` — канат держится у центра → **есть игра**, контент кладётся поверх.
- `DRIFT→DARK` / `DRIFT→LIGHT` — монотонно сваливается в край → крутить `config.js`.
- «концовка» сессии (выжили / погибли-все-V / погибли-все-D) — **контекст, а не вердикт**:
  крах — естественный конец сессии (§7), он сам по себе не говорит о форме шкалы.

### 2. Браузер (играбельная основа)
ES-модули требуют HTTP (не `file://`):
```bash
cd prototype
npm run serve            # python3 -m http.server 8080
# открыть http://localhost:8080
```
Боты ведут обе стороны. «Вселиться»: `1`→D, `2`→V, `0`→отпустить. Движение `WASD`,
выстрел — зажатая мышь (у V выстрел = хил союзника / урон врагу). Тумблер «отзывчивый D»
и график **net(t)** показывают разницу «дрейф против плато» вживую.

## Архитектура

Ключевое решение: **детерминированное сим-ядро (`src/sim`, чистый JS без DOM) отделено
от рендера.** Одно и то же ядро бежит и headless в Node (быстрый прогон каната), и в
браузере на Canvas.

```
src/
  config.js         все числа = placeholder'ы (§11: "насколько" решается прототипом)
  core/             vec2, сид-RNG (детерминизм прогонов)
  sim/              ЯДРО (переносимо): world, entities, darkness(канат §4),
                    economy(§5), spawner(§7), combat(§2,§3), ai(боты D/V/врагов)
  telemetry/        recorder + классификатор формы каната (§12)
  render/           renderer + hud (только браузер, примитивные фигуры)
  browser-main.js   оболочка: цикл, ввод, вселение
tools/run-sim.js    headless-раннер со свипами
```

Каждый модуль помечен ссылками на разделы GDD — причины важнее кода (как и просит документ).

## Первый результат прогона (честно)

С **жадными** ботами (вкладывают очко сразу, как накопят) канат **монотонно сваливается
в тьму** во всех конфигурациях: доход D с убийств стабилен и rate-capped, бонус тёмного
мира — **симметричный** множитель (ускоряет лидера, не меняет баланс), а «премия за
дефицит» из §5 — эффект *толпы* («D много и под огнём»), которого при **одном** D нет.
Раздувание экономики хила V втрое лишь замедляет дрейф, но не разворачивает его.

С **отзывчивым** D (придерживает тьму у сетпоинта — прямая иллюстрация §4: «взять
огромную ноду урона, понимая, что она кладёт моих V под давление?») канат **стабилен**:
net выходит на плато, пара живёт всю сессию.

```
ДЕФОЛТ (жадный):                  ОТЗЫВЧИВЫЙ (--set ai.dDarkStopAt=0.6):
net │      ...:::---===+++***###@   net │   .-+*%@@@@@@@@@@@@@@@@@@@@@@@@
    DRIFT→DARK 4/4                      STABLE 4/4 (выжили всю сессию)
```

**Вывод среза:** ядро *способно* удержать канат (игра есть), но не само по себе —
**только если стимулы реально направляют вложения игрока** (ставка §10). Это смещает
центральный риск с «сходится ли математика» на «заставит ли своекорыстный расчёт D
*притормозить* затемнение». Последнее — проверяемо живой игрой и следующими экспериментами.

## Куда копать дальше (привязка к открытым вопросам §11)

- **§11.1 (сходимость каната):** отзывчивая политика вложений D/V даёт плато — нужен
  поиск режима *колебания* (а не только плато): запаздывания, гистерезис §11.5.
- **Премия за дефицит §5:** проверить с **≥2 D** — манифестируется ли эффект толпы,
  которого нет при 1 D (тогда канат может самокорректироваться и при жадных ботах).
- **§11.2 (пропускная способность хила):** свип `V.healPower` × `enemy.*DarkGain`.
- **§11.4 (сжатие V-кромки):** ввести дальнобоя — пока его нет (один тип врага по §12).

Все рычаги — в `src/config.js`, прогоняются через `--set`.
